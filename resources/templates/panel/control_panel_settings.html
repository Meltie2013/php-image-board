Settings Overview

Use the category list to open a focused settings page, or add a new registry entry when a feature needs a new saved value.

How the settings pages work

  • Each category page is generated from the stored category metadata and the prefix used in the setting key.
  • Built-in settings keep their cleaner labels, descriptions, and safer input controls automatically.
  • Custom settings can still be added here when you need a new registry key before a dedicated UI exists for it.

Recommended workflow

  • Review the category metadata first when the setting group needs a clearer name or description.
  • Add new keys using the matching prefix, such as gallery.example_setting or profile.badges_enabled.
  • Return to that category page and save the final value there once the new entry is visible in the directory.

Advanced Registry Entry

Add or update a raw registry key directly when you need to introduce a new Control Panel-managed setting.

For JSON values, enter valid JSON. Template function allow-lists are validated and normalized before saving. Use Category Settings when you want a custom category title, description, or icon instead of the generic auto-generated group.
